1. Introduction

We are glad you found the new website for SkyPortal and hope you continue to visit us for support of your portal. If you downloaded our software from any site other than http://www.SkyPortal.net, please contact us and let us know as it is in violation of the SkyPortal Licensing Agreement.

SkyPortal is a modular web portal and online community system which includes advanced features such as web-based administration, polls, private/public events calendar, user customizable skins, classifieds, user control panel, links manager, downloads manager, article manager, picture gallery and much more. Easy-to-use and powerful user interface allows members to add news, content, write reviews and share information among other registered users.

This version has many bug fixes, several new security related features, and, above all, has many of the most popular features available to make one of the most feature-packed releases of SkyPortal ever!

To see a list of the new features available for this release, please visit our website at www.SkyPortal.net.

SkyPortal has been designed to be as easy to install as possible. Follow these instructions and you should have no problems. If you do encounter any problems installing SkyPortal, please feel free to contact us.

3. Quick Installation

If you are sure that your hosting service or server will run SkyPortal or have previously used SkyPortal, you can use these steps to quickly get started. For a more detailed explanation please go to the next section.

  1. Extract the files to a local directory on your computer.
  2. Rename the database in the db/ folder.
  3. Modify config.asp as instructed in the file.
  4. Upload the blank database (sp_db2k6.mdb (renamed) for Access database) to a directory that has read/write/modify permission. Use the same path as in the config.asp.
  5. Upload all files to a web accessible directory on your server or hosting account.
  6. Copy the global.asa to your domain root. It will not work in a subfolder.
    If you already have a global.asa in your domain root, open and copy the contents of the SkyPortal global.asa file to your existing global.asa.
  7. The FILES directory needs to have read, write and delete permissions for the ASP.NET, NETWORK, and either the IUSR_ and/or IWAM_ groups in order to use the Upload and Image resizing features.
  8. Open your favorite browser and run site_setup.asp. example: http://www.yourwebsite.com/site_setup.asp
    Follow the instructions provided.
You have now finished setting up SkyPortal on your server. It is highly recommended that you read Post-Install Tasks section.
4. Webhost Installations
3.i. Installing with MS Access Database

Please read this section thoroughly before proceeding with installation.

  1. Extract the files to a local directory on your computer.
  2. Open config.asp in your favorite text editor. example: notepad
    Do not use Microsoft Word, Microsoft WordPad, or Microsoft Frontpage to edit these files as each might corrupt the file.

    • Change the strConnString variable with the correct path to where your database file (*.mdb) is going to be on the server. If you are unsure of which strConnString to use, please use the default one.


    • Change the strWebMaster variable to what you want your admin login name to be. Don't forget the comma after the name. You will use this name when you run SITE_SETUP.ASP

  3. Upload all files (except your database) to a web accessible directory on your server or hosting account.
  4. Copy the global.asa to your domain root. It will not work in a subfolder.
    If you already have a global.asa in your domain root, open and copy the contents of the SkyPortal global.asa file to your existing global.asa.
  5. Open the db folder and rename your database.
    example: my_database.mdb


  6. Upload the blank (empty) database to a directory that the IUSR_ and/or IWAM_ group(s) have read and write permission. Ideally, this directory is not web accessible, to prevent people from downloading your database. This should be the same path as in the strConnString.


  7. The FILES directory needs to have read, write and delete permissions for the ASP.NET, NETWORK, and either the IUSR_ and/or IWAM_ groups in order to use the Upload and Image resizing features.


  8. Open your favorite browser and run site_setup.asp to fill in the setup form.
    example: http://www.yourwebsite.com/site_setup.asp

You have now finished setting up SkyPortal on your server. Please read Post-Install Tasks section.

5. Localhost Installation
5a. IIS Admin using the MMC (Microsoft Management Console)
  • Add the IIS snap-in to the MMC shell to make administration of your web-sites easier
         1. Click Start>Run and type MMC and press OK
         2. On the Console menu, select File > Open and browse to
             C:\Windows\System32\inetsrv.
         3. Select iis.msc and click Open to load the snap-in into your MMC
             – you can now perform all your site administration from within this window
5b. Preparing Localhost for SkyPortal Installation
  • Open Windows Explorer and navigate to C:\Inetpub\wwwroot
    (this is the root of your IIS webserver – you can host multiple websites from here)
  • Create a new folder in wwwroot for your new website (e.g. SkyPortal).
  • Open your MMC - you will now see a file structure in the left-hand side.- Expand the folders
        1. Right-click on “Default Web Site” and select New > Virtual Directory
           which will open the Virtual Directory Creation Wizard.
        2. Click Next and enter a name for your web-site (e.g. SkyPortal).
        3. Click Next
        4. Browse to the directory which you created in step 1 above.
           Select the folder and click OK
        5. Click Next
        6. Select the permissions you wish your site to have – read, write, and
           run scripts is usually enough. Click Next to complete the wizard
5c. Preparing SkyPortal for Localhost installation
  1. Unzip the SkyPortal core into the folder you created in step 1 above
  2. Copy the global.asa to your wwwroot folder. It will not work in a subfolder.
    If you already have a global.asa in your wwwroot folder, open and copy the contents of the SkyPortal global.asa file to your existing global.asa.
  3. In Windows Explorer, navigate to C:\Inetpub\wwwroot\SkyPortal.
  4. Right-click on the db folder and select Properties>Security
  5. Ensure the database and database folder has Read, Write, and Modify permissions set (this must include the Internet Guest Account user pcname\IUSR_pcname). If prompted, ensure changes are propagated to the files and child folders – it is important that the same permisisons are applied to the database.
  6. The FILES directory needs to have read, write and delete permissions for the ASP.NET, NETWORK, and either the IUSR_ and/or IWAM_ groups in order to use the Upload and Image resizing features.
  7. Open the db folder and rename your database
  8. Edit config.asp
        a. Update the strConnString variable with the correct path
            to your database file (*.mdb)
        b.     Change the strWebMaster variable to what you want your admin login
           name to be. Don't forget the comma after the name as you will use this
           name when you run SITE_SETUP.ASP
        c.  Ensure bFso and bAspNet are set to true or false
5d. Setting up SkyPortal
  1. Open Internet Explorer
  2. In the address bar, type http://localhost/SkyPortal/site_setup.asp
    (where SkyPortal is the name for the site you created earlier).
  3. Complete the setup wizard
  4. Take a bow – you’ve done it !
  5. Please read Post-Install Tasks section.
6. Upgrade from MaxWebPortal

This section is only for intermediate users who know exactly what they are doing.
Backup all your files and database before proceeding with the installation.

This version of SkyPortal will detect and upgrade MaxWebPortal v1.3x,
MaxWebPortal v2.1x and upgrade it to the newest version of SkyPortal.

Upgrading from MaxWebPortal v1.3x or MaxWebPortal v2.1x

  1. Backup your entire site, database and all.
  2. Download and unzip the SkyPortal 'core' to your hard drive.
  3. If you are upgrading any or all of the existing Articles, Downloads, Classifieds,
    Forums and/or Pictures. Download them from the SkyPortal website and unzip
    them to your SkyPortal folder.
  4. Edit CONFIG.ASP for your existing database connection, SuperAdmin members,
    LCID, and other settings you wish to set.
  5. Edit fp_custom and make sure the 'include' files are uncommented
    (per module instructions).
  6. Upload all files to your webserver.
  7. Copy the global.asa to your domain root. It will not work in a subfolder.
    If you already have a global.asa in your domain root, open and copy the contents of the SkyPortal global.asa file to your existing global.asa.
  8. The FILES directory also needs to have read, write and delete permissions for the ASP.NET and NETWORK groups in order to use the Upload and Image resizing features.
  9. Copy your avatar folder to your "files" folder
  10. Copy your "files/gallery_images" folder and rename it "pictures".
  11. Run site_setup.asp and follow the instructions to upgrade your database.
  12. Any modules in the list that are NOT selected to upgrade will have their
    database tables deleted along with all data in them.
  13. Login to your site and go to the admin area and run the Avatar Sync to
    populate the new folder path in the database for your site avatars.
  14. If you have active forums, do not set your portal time-offset to a negative number.
  15. Delete any files in your site root that start with "create****.asp".
  16. Delete the contents of the "install" folder.
  17. Delete "site_setup.asp" .


    NOTE: YOUR CURRENT THEME/SKIN WILL NOT WORK WITH SKYPORTAL.

8. Post-Install Tasks - new install only

Once you have successfully installed SkyPortal, log in with the login name you used as the strWebMaster (without the comma at the end) and password you chose, click the Admin Options link located on left menu. Log in again, to access the admin control panel.

For additional security, it is recommended to move your access database file to a private folder and rename the file. example: DogHouse.mdb. This is the same name used in the strConnString variable in the config.asp file. It is also recommended that you delete the site_setup.asp file from your server as well as any files inside of the "install" folder.

 Please take your time and go through every option in the admin control panel.

9. Frequently Asked Questions

Q: I get the error message "Operation Must Use An Updatable Query", how do I fix it?
A: This error is typically encountered when Internet Guest account (IUSR_MACHINE), which is by default part of the "Everyone" group, does not have Write permission on the database. Give read and write permission to the directory where the database file is stored.

Q: I get the error message "Object required: 'dDataCache' ", how do I fix it?
A: This error is typically encountered when the global.asa is not in the domain root folder. It will not work in a subfolder. Copy the global.asa to your domain root. If you already have a global.asa in your domain root, open and copy the contents of the SkyPortal global.asa file to your existing global.asa.

Q: I'm running SkyPortal on a Non-US English server, the dates are not displayed properly. How can I fix it?
A: Open config.asp in your favorite text editor.  Change Session.LCID = 1033 to Session.LCID = 1036 (French). To find the LCID of your country please visit this site.
